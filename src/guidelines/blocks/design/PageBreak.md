# Page Break Block

**Type:** Block  
**Category:** Design  
**Status:** Complete  
**Last Updated:** 2026-02-03

---

## 📋 Overview

The Page Break block separates content into multiple pages. Pagination links appear at the bottom of the content to navigate between these pages.

---

## 🎯 Purpose

Use the Page Break block to:
- Split extremely long articles into manageable chunks.
- Reduce initial page load weight for very content-heavy pages.

---

## 📐 Structure

```
Page Break (comment/separator in editor, pagination on frontend)
```

---

## 💻 Implementation

### **Props/Parameters**

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `className` | `string` | No | `''` | Additional CSS classes |

### **Code Example**

```tsx
import { PageBreak } from '@/components/blocks/design/PageBreak';

<p>Page 1 content...</p>
<PageBreak />
<p>Page 2 content...</p>
```

### **CSS Variables Used**

- `--wp--preset--spacing--40`
- `--wp--preset--spacing--20`

---

## ♿ Accessibility

- **Pagination:** Pagination links must be accessible and clearly indicate current page.
- **Title:** The page title usually remains the same, but the `<title>` tag often appends " - Page 2" for context.

---

## 📱 Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (<640px) | Pagination links wrap if necessary. |

## ✅ Best Practices

- ✅ **DO** use sparingly. Infinite scroll or lazy loading is often preferred in modern UX.
- ✅ **DO** ensure each page has a logical stopping point.

---

## ❌ Common Mistakes

- ❌ **DON'T** split content mid-sentence.
