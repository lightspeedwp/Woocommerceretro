## What Goes Here

**✅ DO store here:**
- Quick reference guides
- Cheat sheets
- Project documentation
- API documentation
- Architecture diagrams
- Step-by-step guides
- Best practices documentation
- Onboarding materials
- Project summaries

**❌ DO NOT store here:**
- Guidelines (use `/guidelines/` instead)
- Reports (use `/reports/` instead)
- Task lists (use `/tasks/` instead)
- Prompts (use `/prompts/` instead)

## Naming Conventions

- Use descriptive, kebab-case names
- Add prefixes for categorization:
  - `quick-ref-*.md` or place in `/quick-references/` — Quick reference guides
  - `guide-*.md` or place in `/guides/` — Step-by-step guides
  - `api-*.md` or place in `/api/` — API documentation
  - `arch-*.md` or place in `/architecture/` — Architecture documentation
- Use UPPERCASE for project-level summaries: `PROJECT-ORGANIZATION-SUMMARY.md`
